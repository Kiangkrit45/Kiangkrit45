Getting Started with React

What is React.js

React.js is an open-source JavaScript library for building user interfaces, specifically for single-page applications. It's used for handling the view layer for web and mobile apps, hence allowing us to create reusable UI components.

Declarative > Imperative

React makes use of JSX standing for JavaScript XML. Improve the predictability and debugability of your code.

Imperative

HTML

<div id="counter">0</div>
<button id="incBin">inc</button>

JavaScript

let count = 0;
const incBin = document.getElementById("incBin");
const counter = document.getElementById("counter");
incBin.onclick = () => counter.textContent = ++count

Declarative

function Counter() {
    const [count, setCount] = useState(0)
    return {
        <>
            <div>{count}</div>
            <button onClick={() => setCount(count + 1)}>inc</button>
        </>
    }
}

Component-Based

Compose encapsulated components that handle their own state to create complicated user interfaces.

It's simple to transmit rich data through your app and keep state off of the DOM because component functionality is written in JavaScript rather than templates.

Write it anywhere you please

Because React.js makes no assumptions about the rest of your technological stack, you can use React to develop new features without having to rewrite old code.

React can also use Node to render on the server and React Native to power mobile apps.

A Basic Component

The render() method in React components takes in inputs and outputs what should be displayed. The JSX syntax, which is similar to XML, is used in the next example.

This is how render() accesses input data that is supplied into the component via this.props.

JSX is not required to use React, but it is preferable to many users.

Example

class HelloMessage extends React.Component {
    render() {
        return (
            <div>
                Hello {this.props.name}
            </div>
            );
        }
    }

ReactDOM.render(
    <HelloMessage name="Taylor" />,
    document.getElementById('hello-example')
    );

Output

Hello Taylor

A Component With State

A component can store internal state data in addition to collecting input data (through this.props) (accessed via this.state). When the state data of a component changes, the rendered markup is changed by re-invoking render().

Example

class Timer extends React.Component {
    constructor(props) {
        super(props);
        this.state = { seconds: 0 };
    }
    tick() {
        this.setState(state => (seconds: state.seconds + 1));
    }
    componentDidMount() {
        this.interval = setInterval(() => this.tick(), 1000);
    }
    componentWillUnmount() { clearInterval(this.interval); }
    render() {
        return <div>Seconds: {this.state.seconds}</div>
    }
}
ReactDOM.render(
    <Timer />,
    document.getElementById('timer-example')
);

An Application

It's possible to create a Todo application using props and state. The example makes use of state to keep track of the current list of items as well as the text provided by the user. Event handlers will be collected and implemented using event delegation, despite the fact that they appear to be rendered inline.

Example

class TodoApp extends React.Component {
    constructor(props) {
        super(props);
        this.state = { items: [], text: '' };
        this.handleChange = this.handleChange.bind(this);
        this.handleSubmit = this.handleSubmit.bind(this);
    }
    render() {
        return (
        <div>
            <h3>TODO</h3>
            <TodoList items={this.state.items} />
            <form onSubmit={this.handleSubmit}>
            <label htmlFor="new-todo">What needs to be done?</label>
            <input
                id="new-todo"
                onChange={this.handleChange}
                value={this.state.text}
            />
            <button>Add #{this.state.items.length + 1}</button>
            </form>
        </div>
        );
    }
    handleChange(e) {
        this.setState({ text: e.target.value });
    }

    handleSubmit(e) {
        e.preventDefault();
        if (this.state.text.length === 0) {
        return;
        }
        const newItem = {
        text: this.state.text,
        id: Date.now()
        };
        this.setState(state => ({
        items: state.items.concat(newItem),
        text: ''
        }));
    }
}
class TodoList extends React.Component {
    render() {
        return (
        <ul>
            {this.props.items.map(item => (
                <li key={item.id}>{item.text}</li>
            ))}
        </ul>
        );
    }
}
ReactDOM.render(
    <TodoApp />,
    document.getElementById('todos-example')
);

A constituent Using Third-Party Plugins

Using React to communicate with other libraries and frameworks. To convert the <textarea> in this example, remarkable, an external Markdown library, is used.

Example


class MarkdownEditor extends React.Component {
    constructor(props) {
        super(props);
        this.md = new Remarkable();
        this.handleChange = this.handleChange.bind(this);
        this.state = { value: 'Hello, **world**!' };
    }
    handleChange(e) { this.setState({ value: e.target.value }); }
    getRawMarkup() { return { __html: this.md.render(this.state.value) }; }
    render() {
        return (
        <div className="MarkdownEditor">
            <h3>Input</h3>
            <label htmlFor="markdown-content">Enter some markdown</label>

            <textarea
                id="markdown-content"
                onChange={this.handleChange}
                defaultValue={this.state.value}
            />
            <h3>Output</h3>
            <div
                className="content"
                dangerouslySetInnerHTML={this.getRawMarkup()}
            />
        </div>
        );
    }
}
ReactDOM.render(
    <MarkdownEditor />,
    document.getElementById('markdown-example')
);
